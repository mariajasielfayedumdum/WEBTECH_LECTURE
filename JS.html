Javascript

    • Developed circa 1995 by Brendan Eich at Netscape Communications as the scripting language for the Netscape Navigator Browser
    • Formerly called Mocha, then LiveScript, then JavaScript
    • Standardized by ECMA International as ECMAScript
    • Latest version: JavaScipt 1.8.1, ECMAScript 5 [ECMA-262 5th Edition]
    • Common version: JavaScript 1.5, Jscript 5.5, ECMAScript v3 [ECMA-262 3rd Edition]
    • JavaScript Frameworks:
      • script.aculo.us, JQuery, MooTools, Prototype, Dojo Toolkit, etc.
    • Linked/Embedded in web pages using the script element
      • Linked:
        script type="text/javascript" src="scripts.js"
      • Embedded (either in the head or the body element):
        script type="text/javascript"
        ...
        noscript
    • Javascript + DOM/BOM + CSS + XHTML = DHTML
    • Javascript code in XHTML pages can be executed "on the fly" as the document is rendered (i.e. code outside of functions executes as the script element is encountered); in most cases though, Javascript code is executed in response to document events (e.g. clicking a page element).
    • Basic language features:
      1. Paradigm:
        • Object-oriented (prototype-based), functional, imperative scripting language
      2. Java / C-like syntax:
        • Implicit semicolon insertion for statement termination
        • Identifiers are alphanumeric, _, and $ characters
        • Single-line (//) or (/**/) comments
      3. Type system and variable scoping rules:
        • Dynamic (aka loose or weak) typing
        • Global (aka top-level) or local scopes
        • Data types:
          1. Primitive types:
            • Numbers (decimal, hexadecimal notation)
            • Booleans (true, false)
            • Strings (Single or Double quote delimited)
            • Undefined and Null
          2. Composite (object) types:
            • Core Javascript Objects:
            • Primitive types:
              1. Object, Number, Boolean, String, Date, Math, Global, RegExp, Error
              2. Arrays (Array)
              3. Functions (Function, Arguments)
            • DOM Objects
              • Anchor, Applet, Attr, Comment, DOMException, DOMImplementation, Document Fragmenet, Element, Event, Form, Image, Input, Layer, Link, Node, Option, Select, Style, Text, Textarea
        • Keywords
          • break, case, catch, continue, default, delet, do, else, finally, for, function, if, in, instanceof, new, return, switch, this, throw, try, typeof, var, void, with
        • Reserved words (currently unused)
          • abstract, boolean, byte, char, class, const, debugger, float
        • Statements and control structures
          1. var
            • Used to declare global/local scoped variables
          2. if-else
            • Condition expressions having values of o, "", null, and undefined evaluate to false
          3. switch-case-default-break
            • allows any expression type to be used as the switch expression
            • case labels may be of different types
            • case labels may be expressions
            • case execution fails-through, unless terminated by a break
          4. while, do-while, for, for-in, break, continue
            • for while and do-while, false condition expressions similar to if-else
            • for-in used for property enumeration
            • allows labeled break/continue
          5. try-catch-finally, throw
            • throw and catch can handle any expression type
          6. function, return
            • Javascript functions are similar to Java method except for the following differences
              1. No return value type is specified, and return is optional
              2. Functions may return a value on one invocation and not return a value (i.e. have an undefined return value) on another invocation
              3. Functions may return different types of values on different invocations
              4. Functions can be invoked with an arbitrary number of arguments, regardless of the actual parameters specified in the function definition (the Arguments object can be used to access unnamed arguments passed to the function invocation)
              5. Functions are first-class objects
          7. with
            • Used to access object properties without having expicitly qualify the property with the object name(serves as shorthand notation for accessing object properties, at the expense of program readability).
    • Asynchronous JavaScript and XML (AJAX)
      • Synchronous Request
        • Recommended for few small requests. JavaScript will not continue to execute, until the server response is ready. If the server is busy or slow, the application will hang or stop.


        var xhr = new XMLHttpRequest();
        xhr.open(“GET”, “somepage.txt”, false);
        xhr.send(null);
        if(xhr.status==200){
        var str = xhr.responseText;
        }

      • Asynchronous Request
        • JavaScript does not have to wait for the server response. It can execute other scripts while waiting for server response, and deal with the response when the response is ready.


        var xhr = new XMLHttpRequest();
        xhr.open(“POST”, “products.php”, true);
        xhr.setRequestHeader(“Content-Type”, “application/x-www-form-urlencoded”);

        xhr.onreadystatechange = function(){
        if(this.readyState == 4 && this.status == 200){
        var hdrDate = this.getResponseHeader(“Date”);
        var xml = this.responseXML; }
        }

        xhr.send(“productID=1234&catID=5678”);

      • Three important properties of the XMLHttpRequest Object
        • onreadystatechange
          • Stores a function to be called automatically each time readyState changes
        • readyState
          • 0 : request not initialized
          • 1 : server connection established
          • 2 : request received
          • 3 : processing request
          • 4 : request finished and response is ready o status

    • Java EE
      • Java EE is an open, standards-based development and deployment platform for creating distributed, transactional, reliable, secure, multitiered, web-based, server-centric, component-based enterprise applications
      • Java EE Application Model
        1. Java programming language, Java Virtual Machine (JVM)
        2. Java EE Components
          • Java EE Clients
            • Application Clients, Applets (embedded in web clients)
          • Web Components
            • Servlets, JavaServer Pages (JSP), JavaServer Faces (JSF)
          • Enterprise JavaBeans (EJBM)
        3. Java EE Containers
          • Client containers, web container, EJB container
        4. Java EE Server
      • Java EE Web Application
        • Collection of resources installed under a specific subset of the URL namespace of a web application server compliant with the Java EE Specification (e.g. Apache‟s Tomcat, Apache‟s Geronimo, Sun Microsystems‟ Glassfish, IBM‟s WebSphere, etc.)
        • Resources
          1. Static resources: web pages, images, stylesheets, etc. (serves as is)
          2. Dynamic resources: servlets, JSPs
          3. Miscellaneous resources: business object classes (e.g. Java Beans, EJB), support libraries, etc.
        • XML-formatted descriptor and configuration files
          • web.xml, application.xml, context.xml, etc.
        • Organized into a standard hierarchical structure and typically packaged and deployed as WAR or EAR files
      • Java EE APIs
        1. Enterprise JavaBeans Technology
        2. Java Servlet Technology
        3. Java Server Pages
        4. Java Server Pages Standard Tag Library
        5. Java Server Faces
        6. Java Msg Service API
        7. Java Transaction API
        8. JavaMail API
        9. JavaBeans Activation Framework
        10. Java API for XML Processing
